# Bug audit — 2026-05-03

Multi-agent bug hunt across `net/crates/net/` (~117k LOC, 158 Rust files). Eight parallel agents each took a subsystem and reported real, demonstrable defects (not style or speculation). Findings sorted by severity.

## Critical

1. **`adapter/net/redex/disk.rs:1322` — `compact_to` is non-atomic on Windows.** `fsync_dir` is a no-op on `cfg(not(unix))`. Combined with the cross-file rename sequence (idx + dat + ts), a power loss can leave the directory pointing at mismatched inodes. Fix: use `MoveFileExW` with `MOVEFILE_WRITE_THROUGH`.

2. **`bus.rs:686-711` — Shutdown await order deadlocks.** `remove_shard_internal` awaits `workers.batch.await` before `workers.drain.await`, but the comment at lines 514-520 says drain must finish first so the batch worker's `recv()` can return `None`. Drain still owns a sender clone. Fix: swap the await order to drain → batch.

3. **`ffi/mod.rs:1322-1331` (`net_poll_ex`) — Panic across FFI on OOM.** `event.id.as_bytes().to_vec().into_boxed_slice()` and `event.raw.to_vec().into_boxed_slice()` can panic mid-loop, leaking earlier `Box::into_raw`s + the `std::alloc::alloc(layout)` array. No `catch_unwind`. Fix: wrap entry points in `catch_unwind`, or use `try_reserve`.

## High

4. **`adapter/net/redex/disk.rs:1356-1413` — Temp-file leak on `compact_to` failure.** Cleanup at 1411-1413 runs *after* `open_or_poison(...)?`/`clone_or_poison(...)?` (1356-1358, 1396-1398). On any post-rename reopen failure the three placeholder files in `/tmp` leak forever.

5. **`adapter/net/cortex/adapter.rs:486-499` — Watermark advances past unfolded events under `Stop` policy.** `recoverable_decode` causes `folded_through_seq.store(seq)` for events whose state mutation never landed; `wait_for_seq(seq)` then returns true incorrectly. Combined with snapshot-restore (#6) this produces permanent state divergence.

6. **`adapter/net/cortex/adapter.rs:374-386` + `tasks/adapter.rs:340-344` — Snapshot persists `last_seq` for skipped events.** When (5) fires, `snapshot()` returns `last_seq = Some(N)` for events whose state was never applied; restore tails from `N+1` and the skipped seq is never re-attempted. Violates "log is the source of truth."

7. **`adapter/net/cortex/meta.rs:135-137` — Checksum doesn't cover meta header.** `compute_checksum` hashes only `tail`. A bit-flip in `dispatch` byte (e.g. `STORED` → `DELETED`) is undetected and silently re-routes the event to the wrong fold arm.

8. **`adapter/net/state/log.rs:175-211` (`prune_through`) — `base_link` not updated on partial prune.** Anchor only refreshes when `events.is_empty()` post-prune. After a partial prune followed by a full prune, `base_link` points at the first prune anchor, not the current head — chain validation breaks.

9. **`adapter/net/redex/file.rs:1100-1126` (`sweep_retention`) — In-memory index corrupted on panic between drain and renormalize.** Lines 1100-1102 evict prefix; 1118-1126 rewrite `payload_offset`. A panic between them leaves rebased `base_offset` with absolute offsets — every subsequent read misses. Fix: build the renormalized index in a temp `Vec`, then atomically replace.

10. **`adapter/net/redex/disk.rs:251-261, 305-311` — Truncate-on-recovery without `sync_all`.** `set_len` for torn-tail repair has no `sync_all`/`fsync_dir`; a crash before the next durable write reverts the recovery and re-reads the same torn bytes.

11. **`adapter/net/redex/disk.rs:735, 951` — Best-effort rollback silently swallows errors.** `if let Ok(f) = OpenOptions::new().write(true).open(...)` — if the rollback open fails, orphan dat bytes / extra idx records persist. Subsequent appends produce permanent dat/idx divergence.

12. **`adapter/net/compute/registry.rs:71-74` (`replace`) — Not safe for stateful in-flight work.** A caller holding an old `Arc<Mutex<DaemonHost>>` from `get_arc` keeps mutating the old host while new traffic hits the new one; old chain's writes are dropped. `replica_group::on_node_failure`, `fork_group::on_node_failure`, and `standby_group::on_node_failure` all use this pattern without quiescing.

13. **`adapter/net/compute/orchestrator.rs:1301-1323` (`abort_migration_with_reason`) — Doesn't propagate to `MigrationSourceHandler`.** Source-side `migrations` map retains the entry; `is_migrating()` stays true, `buffer_event` keeps buffering into an undrained vector, retries trip `AlreadyMigrating`.

14. **`adapter/net/compute/standby_group.rs:354-389` — Replaced standby marked healthy with `synced_through = 0`.** If the active fails before next `sync_standbys`, `promote()` (`max_by_key(synced_through)`) may pick the fresh zero-state standby and lose all pre-buffer state. Fix: keep replaced standby unhealthy until after a successful sync, or filter promote candidates to `last_sync.is_some()`.

15. **`adapter/net/compute/bindings.rs:117-118` — Unbounded `Vec::with_capacity` from attacker u32.** `count = cursor.get_u32_le() as usize; Vec::with_capacity(count)` — peer-supplied snapshot can declare ~4B entries, ~96 GiB allocation before truncation checks. Fix: bound by `data.len() / MIN_BINDING_SIZE`.

16. **`adapter/net/cortex/tasks/watch.rs:64-86`, `memories/watch.rs:95-117` — Doc says strict bounds, code is inclusive.** `created_after` / `created_before` etc. published as `>` / `<`; underlying `…FilterSpec::matches` uses `>=` / `<=`. Strict-bound consumers receive boundary events.

17. **`adapter/net/reliability.rs:308-323` (`on_send`) — Silent reliability drop.** When `pending.len() >= max_pending`, the oldest *unacknowledged* packet is `pop_front`-ed; subsequent NACK can never recover that seq because the entry is gone (`on_nack` at 408-417 finds nothing to match). Fix: backpressure callers; don't drop tracking for in-flight packets.

18. **`adapter/net/router.rs:559-580` (`NetRouter::start`) — No re-entry guard.** Calling twice spawns two competing dequeue loops on the same `FairScheduler`, racing `pop()`. Fix: `compare_exchange` on `running`.

19. **`adapter/net/behavior/safety.rs:1378-1407` — AuditOnly mode silently drops violation logs.** `check_rate_limits` only logs the global-RPM violation when `mode == Enforce`; AuditOnly's documented "log violations but don't block" doesn't actually log. Fix: log unconditionally; gate only the `return Err` on enforce.

20. **`adapter/net/behavior/proximity.rs:316` (`update_from_pingwave`) — Worse path overwrites better.** Updates whenever `pw.seq > self.last_seq` regardless of hop count. A flooded high-seq pingwave through a long route demotes the cached direct route. Fix: separate freshness (always take latest seq) from path quality (only update `hops`/`addr`/`latency_us` when `new_hops <= self.hops`).

21. **`adapter/net/subnet/gateway.rs:144-152` — `ParentVisible` over-permissive direction.** Predicate accepts both `dest.is_ancestor_of(source)` and `source.is_ancestor_of(dest)`; the second clause leaks parent-region traffic *down* into descendants, contradicting the documented "visible to ancestors" framing. Tests cover only child→parent and sibling→sibling.

## Medium

22. **`adapter/net/cortex/adapter.rs:122-141` (`wait_for_seq`)** — When `start_seq == 0` and no events fold, the `u64::MAX` sentinel + always-true `running` makes `wait_for_seq(0)` block forever. Direct `CortexAdapter` users on an empty file hang.

23. **`adapter/net/cortex/memories/fold.rs:67-86` — Asymmetric upsert semantics under `STORED`.** Existing-id store partial-updates (preserves `pinned`/`created_ns`, overwrites the rest); missing-id inserts with `pinned: false`. The `MemoriesAdapter::store` doc reads "Store a new memory" — silent overwrite is surprising. Combined with absence of tombstones (`memories/fold.rs:115`), a STORED→DELETED→STORED sequence resurrects deleted entries.

24. **`adapter/net/cortex/memories/query.rs:77-86` — Empty-vec filter is a footgun.** `Some(vec![])` for `require_any_tag` excludes everything (`any` over empty = false → reject); `Some(vec![])` for `require_all_tags` excludes nothing (`all` over empty = true). UI forms emitting empty multi-selects break silently.

25. **`adapter/net/failure.rs:783-805` (`RecoveryManager::on_failure`)** — `failed_nodes.insert(...)` overwrites existing `FailedNodeState`, resetting `failed_at` and `retry_count` to 0. Flapping peers never hit `max_retries`. Fix: `entry().or_insert(...)` and bump `retry_count`.

26. **`adapter/net/session.rs:543-565` (`evict_idle_streams`) — LRU vs concurrent open race.** `streams.iter().min_by_key(...)` then `remove` is non-atomic. A freshly-opened stream can be torn down between selection and removal. Fix: `remove_if` with a freshness predicate.

27. **`adapter/net/mod.rs:451` — Handshake retry has no upper bound.** `tokio::time::sleep(Duration::from_millis(100 * attempt as u64))` over `MAX_HANDSHAKE_RETRIES = 1024` ⇒ ~14 hours total, last attempt sleeps ~102s. Fix: cap the per-attempt sleep at e.g. 5s.

28. **`adapter/net/redex/file.rs:402-462` (`append`) — Observer inconsistency.** `next_seq` is bumped before `segment.append`, so a reader between (`len()`, `next_seq()`) sees `len + 1 > next_seq_seen`. Fix: maintain a separate `committed_seq` for observers.

29. **`bus.rs:1337` — `JoinHandle` errors silently dropped.** `let _ = futures::future::join_all(drains).await;` — a panicked drain worker isn't logged unless the runtime's panic hook is configured (default Tokio doesn't). Stranded events.

30. **`adapter/net/identity/origin.rs:42-43` — `origin_hash: u32` collision floor.** ~65k peer birthday-collision; cross-channel accounting keyed by `origin_hash` aliases distinct entities. Fix: rename to `origin_tag` to dispel identity reading, or widen to u64 in a wire bump.

31. **`shard/mapper.rs:1024` (`finalize_draining`) — Wrong atomic ordering.** Reads `pushes_since_drain_start` with `Relaxed` while the field's docstring requires `Acquire` to pair with the writer's `SeqCst` reset.

32. **`adapter/net/identity/token.rs:565-594` (`insert_unchecked`) — Slot-cap race unbounded.** `contains_key` then `entry()` overshoot is bounded by concurrent calls, not shards. Fix: `entry().or_insert_with()` then drop on overflow.

33. **`adapter/net/behavior/diff.rs:241-243` — `to_bytes` returns `Vec::new()` on cap-violation.** Indistinguishable from a legitimate empty diff. Senders silently transmit zero bytes; receiver drops. Fix: deprecate `to_bytes` in favor of `try_to_bytes`.

34. **`adapter/net/contested/propagation.rs:62-106` — Lossy `f64 → u64` poisons EWMA.** A pathological RTT clamps `per_hop` to `u64::MAX`, locking `base_hop_latency_nanos` near max permanently. Combined with the `multiplier == 0.0` check (line 96) which doesn't catch NaN.

35. **`adapter/net/contested/correlation.rs:148` — `Instant` subtraction can panic.** `now - self.config.correlation_window` panics if `correlation_window > now.elapsed()` (configurable to hours).

36. **`adapter/net/contested/superposition.rs:107-111` — `target_replayed` only advances from `Superposed`.** If called while in `Spreading` (target catches up before `advance(Replay)`), phase stalls in `Spreading` forever; `ReadyToCollapse` never fires.

37. **`adapter/net/contested/observation.rs:62-73, 115` — Unbounded `HashMap` growth + `seq_diff_sum` overflow.** No eviction in `last_observed`/`estimated_delay`; `u64::abs_diff` accumulator overflows on long chains. Fix: LRU + `saturating_add`.

38. **`adapter/net/compute/orchestrator.rs:1177-1182` (`on_replay_complete`) — Forged `target_head` with `parent_hash: 0`.** Synthetic link doesn't match any real hash; downstream verifiers can't reconcile.

39. **`adapter/net/compute/migration_target.rs:244-261` (`buffer_event`) — No phase guard.** Inserts/delivers post-cutover; combined with normal-path delivery yields duplicate execution.

40. **`adapter/net/compute/migration_source.rs:107-145` (`start_snapshot`) — Snapshot side-effect race.** `contains_key` → `entry()` window allows two concurrent snapshots of the same origin via user `MeshDaemon::snapshot()`. Trait API doesn't enforce idempotency.

41. **`adapter/net/compute/standby_group.rs:213-253` (`sync_standbys`)** — Partial-failure leaves standbys at mixed states with `synced_through` unchanged and buffer not cleared; subsequent `promote()` may double-deliver.

42. **`adapter/net/compute/registry.rs:91-96` (`unregister`)** — In-flight `Arc<Mutex<DaemonHost>>` clones survive past unregister; a fast `register` of the same origin produces two parallel Arcs locking independently, splitting writes.

43. **`adapter/net/state/causal.rs:248` — Doc says `CausalLink: 24 bytes`, constant is 28.** Wire-format spec contradicts code; external parsers reading per the doc go off-by-4.

44. **`adapter/net/contested/partition.rs:241-246` — `NaN >= threshold` blocks healing.** When `other_side.is_empty()`, ratio is NaN and partition can't heal automatically.

45. **`adapter/net/redex/disk.rs:1196` — `saturating_sub(dat_base) as u32` for stale heap entry silently writes offset 0.** Currently safe by `dat_base` derivation, but masks corruption rather than failing.

46. **`adapter/net/traversal/portmap/natpmp.rs:537-606` (`remove`)** — Silent early returns on `bind`/`connect`/`send` failures; operator never sees that the prior mapping wasn't revoked.

47. **`consumer/merge.rs:384` — Per-shard cap clamps cursor advancement.** When `unclamped_per_shard > PER_SHARD_FETCH_CAP`, cursor rolls back instead of advancing to the last fetched event id; single-shard requests with filters can stall in re-fetch loops.

## Audit notes

- Three subsystems are notably well-defended: `crypto.rs` (nonce/replay/HKDF/AEAD all sound), `pool.rs` (TX nonce coordination correct), and most `FailureDetector`/`CircuitBreaker` paths.
- `mesh.rs` (~7953 lines) was not fully read in this pass — only spot-checks. Worth a dedicated follow-up.
- The codebase has aggressive regression tests (`test_regression_*`) that pin many historical bugs; the findings above are not covered by those.
- Triage priority: #1, #2, #3 (Critical), then #4 (temp-file leak), #5–#7 (cortex watermark/checksum durability), #12 (`registry::replace`), #15 (FFI DoS).

## Follow-up pass — 2026-05-03

Second multi-agent sweep over `bus.rs`, `event.rs`, `timestamp.rs`, `shard/`, `consumer/`, `ffi/`, `adapter/` top-level, and `adapter/net/` top-level. New findings only — duplicates from the first pass omitted (e.g. `shard/mapper.rs:1024` finalize_draining ordering = existing #31).

### High

51. **`ffi/cortex.rs` (~30 sites: lines 207, 275, 323, 339, 354, 366, 388, 417, 487, 519, 571, 597, 622, 643, 667, 836, 871, 908, 970, 1000, 1062, 1103, 1132, 1154, 1175, 1196, 1390, 1421, 1453) — Handle-lifetime contract is unenforced.** Unlike `ffi/mod.rs`, the cortex FFI uses bare `&*handle` with no `FfiOpGuard`/`active_ops` analog. A caller that races `net_redex_free` against `net_redex_file_append` produces use-after-free: `Box::from_raw` in `_free` deallocates while another thread holds `&*handle`. The Go cgo SDK is exactly this multi-thread pattern. Fix: port the `active_ops` + `bus_taken` CAS protocol from `ffi/mod.rs`.

52. **`ffi/mesh.rs:498-505, 1131-1138, et al. — Same lack of shutdown/free synchronization across all mesh handles.** `net_mesh_free` / `net_mesh_stream_free` / `net_identity_free` / `net_redis_dedup_free` are unsynchronized vs concurrent uses of the same handle.

53. **`ffi/mesh.rs:1078-1079` — `MeshStreamHandle._node` keeps the node alive but not the handle box.** `_node: Arc<MeshNode>` pins the node, but `net_mesh_free(node_handle)` deallocates the handle's `Box` while `net_mesh_send` is dereferencing `&*node_handle` (e.g. through `handles_match` / `Arc::ptr_eq`). UAF on the handle pointer.

### Medium

54. **`bus.rs:1056-1106` (`flush` Phase 2) — Progress condition can return early under concurrent ingestion.** Gate uses the *global* `events_dispatched` counter; `target_ingested` is snapshotted at flush start, but `events_dispatched` keeps incrementing for events ingested *after* flush() began. A workload where post-flush traffic lands on shard A while pre-flush traffic sits in shard B's mpsc can satisfy `dispatched + dropped_during_flush ≥ target_ingested` while pre-flush events are still in flight. Defeats the documented delivery-barrier contract. Fix: snapshot `events_dispatched_at_start` and gate on the delta, or use per-event sequence numbers.

55. **`bus.rs:1437-1446` (`manual_scale_down`) — Leaks orphan shards into bus tables.** `mapper.finalize_draining()` returns *every* shard whose grace expired this tick. `manual_scale_down` only calls `remove_shard_internal` for IDs in its own `target` set, so any concurrently-finalized shard becomes `Stopped` while its workers, sender, and routing entry remain in `batch_workers` / `batch_senders` / routing tables.

56. **`ffi/mod.rs:1075-1081` — Concurrent `net_shutdown` callers race the `bus_taken` swap.** The drain protocol guards in-flight FFI ops but not concurrent shutdown callers themselves. A second/third caller can return `Success` while the first is still inside `runtime.block_on(bus.shutdown())` — falsely signaling completion of an in-progress shutdown. The doc comment "C contract guarantees not called concurrently with itself" admits this; nothing enforces it.

57. **`ffi/cortex.rs` and `ffi/mesh.rs` (every `runtime().block_on(...)` site: cortex.rs:419, 670, 910, 1199, 1455; mesh.rs:475, 639, 660, 676, 698, 810, 829, 902, 1209, 1242, 1277, 1360, 1689) — No `Handle::try_current()` guard.** `ffi/mod.rs:21-53` documents the contract; the cortex/mesh modules don't enforce it. A `block_on` panic unwinds across `extern "C"` → UB. `catch_unwind` shim is missing.

58. **`adapter/jetstream.rs:267-380` (`on_batch`) — Partial-failure can produce duplicate publishes.** Mid-batch publish error drops in-flight `PublishAckFuture`s but bytes are already on the wire; retry re-publishes. `Nats-Msg-Id` dedups *only within the dedup window* (default 2 min). Slow caller / long retry backoff lands the same event at two distinct seqs. Comment at 304-308 calls this "safe" but downstream `seq` consumers see duplicates.

59. **`adapter/jetstream.rs:336-371` — Async-cancellation hazard mid-loop.** No timeout wraps `publish_with_headers`. Tokio cancellation drops `acks` without awaiting; bytes already sent persist. Same wire/local-cancel split as #58. Compare `redis.rs:396` which does wrap.

60. **`adapter/redis.rs:316-405` — Pipeline timeout duplicate hazard documented but unfixed by default.** Lines 376-394 document the race. The `dedup_id` mitigation depends on the consumer running `RedisStreamDedup`, which is *not* wired up by default — Go consumers that skip the helper see duplicates.

61. **`adapter/net/mesh.rs:4191` — Lone non-saturating `hop_count += 1` on attacker-controlled byte.** Every other hop-count increment in the crate uses `saturating_add(1)` (`swarm.rs:122`, `route.rs:254`). Bounded today by the `< MAX_CAPABILITY_HOPS - 1` (15) guard above it, but one constant change from a debug-panic / release-wraparound on attacker-controlled input. One-line hardening.

### Lower-priority

- **`shard/batch.rs:73`** — `let window_start = now - self.config.velocity_window;` panics on `Instant - Duration` underflow. On Windows `Instant` is QPC-relative to boot; with a velocity_window of seconds and a process started immediately after boot, this aborts the batch worker task. Use `checked_sub`.
- **`shard/mapper.rs:875-904` (`drain_specific`)** — Doesn't bump `last_scaling`. `scale_down` always writes `last_scaling = Some(Instant::now())` (mapper.rs:976) so a follow-up `scale_up` is gated by cooldown; `drain_specific` removes a shard from Active without touching `last_scaling`, so `drain_specific(id) → scale_up(N)` bypasses the cooldown floor.
- **`bus.rs:2060-2064` (suspect)** — Drain worker `mem::replace` swaps `scratch` *before* the `sender.send(batch).await`. If the channel closes while the await is parked, the batch is silently dropped and the loop breaks — events never reach the adapter. Safe under documented shutdown ordering today; fragile to reorder (the activate-failure rollback path at 487-573 is the easy way to expose it).
- **`adapter/net/linux.rs:138-139` (suspect)** — `iov_base: packet.as_ptr() as *mut _` launders `*const u8` to `*mut` for `sendmmsg`. Sound (kernel reads only, `&[Bytes]` keeps storage alive) but the kind of pointer-provenance laundering Miri flags.
- **`adapter/net/pool.rs:198, 265`** — `(payload.len() - 16) as u16` truncation is safe under current `MAX_PAYLOAD_SIZE = 8112`, but unchecked. If anyone ever raises the cap past `u16::MAX + 16` without revisiting these lines, the wire `payload_len` silently truncates and the receiver mis-frames. Add `debug_assert!(payload.len() - 16 <= u16::MAX as usize)`.
