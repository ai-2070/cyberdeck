# Bug audit ‚Äî 2026-05-03

Multi-agent bug hunt across `net/crates/net/` (~117k LOC, 158 Rust files). Eight parallel agents each took a subsystem and reported real, demonstrable defects (not style or speculation). Findings sorted by severity.

## Critical

1. **`adapter/net/redex/disk.rs:1322` ‚Äî `compact_to` is non-atomic on Windows.** `fsync_dir` is a no-op on `cfg(not(unix))`. Combined with the cross-file rename sequence (idx + dat + ts), a power loss can leave the directory pointing at mismatched inodes. Fix: use `MoveFileExW` with `MOVEFILE_WRITE_THROUGH`.

2. **`bus.rs:686-711` ‚Äî Shutdown await order deadlocks.** `remove_shard_internal` awaits `workers.batch.await` before `workers.drain.await`, but the comment at lines 514-520 says drain must finish first so the batch worker's `recv()` can return `None`. Drain still owns a sender clone. Fix: swap the await order to drain ‚Üí batch.

3. **`ffi/mod.rs:1322-1331` (`net_poll_ex`) ‚Äî Panic across FFI on OOM.** `event.id.as_bytes().to_vec().into_boxed_slice()` and `event.raw.to_vec().into_boxed_slice()` can panic mid-loop, leaking earlier `Box::into_raw`s + the `std::alloc::alloc(layout)` array. No `catch_unwind`. Fix: wrap entry points in `catch_unwind`, or use `try_reserve`.

## High

4. **`adapter/net/redex/disk.rs:1356-1413` ‚Äî Temp-file leak on `compact_to` failure.** Cleanup at 1411-1413 runs *after* `open_or_poison(...)?`/`clone_or_poison(...)?` (1356-1358, 1396-1398). On any post-rename reopen failure the three placeholder files in `/tmp` leak forever.

5. **`adapter/net/cortex/adapter.rs:486-499` ‚Äî Watermark advances past unfolded events under `Stop` policy.** `recoverable_decode` causes `folded_through_seq.store(seq)` for events whose state mutation never landed; `wait_for_seq(seq)` then returns true incorrectly. Combined with snapshot-restore (#6) this produces permanent state divergence.

6. **`adapter/net/cortex/adapter.rs:374-386` + `tasks/adapter.rs:340-344` ‚Äî Snapshot persists `last_seq` for skipped events.** When (5) fires, `snapshot()` returns `last_seq = Some(N)` for events whose state was never applied; restore tails from `N+1` and the skipped seq is never re-attempted. Violates "log is the source of truth."

7. **`adapter/net/cortex/meta.rs:135-137` ‚Äî Checksum doesn't cover meta header.** `compute_checksum` hashes only `tail`. A bit-flip in `dispatch` byte (e.g. `STORED` ‚Üí `DELETED`) is undetected and silently re-routes the event to the wrong fold arm.

8. **`adapter/net/state/log.rs:175-211` (`prune_through`) ‚Äî `base_link` not updated on partial prune.** Anchor only refreshes when `events.is_empty()` post-prune. After a partial prune followed by a full prune, `base_link` points at the first prune anchor, not the current head ‚Äî chain validation breaks.

9. **`adapter/net/redex/file.rs:1100-1126` (`sweep_retention`) ‚Äî In-memory index corrupted on panic between drain and renormalize.** Lines 1100-1102 evict prefix; 1118-1126 rewrite `payload_offset`. A panic between them leaves rebased `base_offset` with absolute offsets ‚Äî every subsequent read misses. Fix: build the renormalized index in a temp `Vec`, then atomically replace.

10. **`adapter/net/redex/disk.rs:251-261, 305-311` ‚Äî Truncate-on-recovery without `sync_all`.** `set_len` for torn-tail repair has no `sync_all`/`fsync_dir`; a crash before the next durable write reverts the recovery and re-reads the same torn bytes.

11. **`adapter/net/redex/disk.rs:735, 951` ‚Äî Best-effort rollback silently swallows errors.** `if let Ok(f) = OpenOptions::new().write(true).open(...)` ‚Äî if the rollback open fails, orphan dat bytes / extra idx records persist. Subsequent appends produce permanent dat/idx divergence.

12. **`adapter/net/compute/registry.rs:71-74` (`replace`) ‚Äî Not safe for stateful in-flight work.** A caller holding an old `Arc<Mutex<DaemonHost>>` from `get_arc` keeps mutating the old host while new traffic hits the new one; old chain's writes are dropped. `replica_group::on_node_failure`, `fork_group::on_node_failure`, and `standby_group::on_node_failure` all use this pattern without quiescing.

13. **`adapter/net/compute/orchestrator.rs:1301-1323` (`abort_migration_with_reason`) ‚Äî Doesn't propagate to `MigrationSourceHandler`.** Source-side `migrations` map retains the entry; `is_migrating()` stays true, `buffer_event` keeps buffering into an undrained vector, retries trip `AlreadyMigrating`.

14. **`adapter/net/compute/standby_group.rs:354-389` ‚Äî Replaced standby marked healthy with `synced_through = 0`.** If the active fails before next `sync_standbys`, `promote()` (`max_by_key(synced_through)`) may pick the fresh zero-state standby and lose all pre-buffer state. Fix: keep replaced standby unhealthy until after a successful sync, or filter promote candidates to `last_sync.is_some()`.

15. **`adapter/net/compute/bindings.rs:117-118` ‚Äî Unbounded `Vec::with_capacity` from attacker u32.** `count = cursor.get_u32_le() as usize; Vec::with_capacity(count)` ‚Äî peer-supplied snapshot can declare ~4B entries, ~96 GiB allocation before truncation checks. Fix: bound by `data.len() / MIN_BINDING_SIZE`.

16. **`adapter/net/cortex/tasks/watch.rs:64-86`, `memories/watch.rs:95-117` ‚Äî Doc says strict bounds, code is inclusive.** `created_after` / `created_before` etc. published as `>` / `<`; underlying `‚Ä¶FilterSpec::matches` uses `>=` / `<=`. Strict-bound consumers receive boundary events.

17. **`adapter/net/reliability.rs:308-323` (`on_send`) ‚Äî Silent reliability drop.** When `pending.len() >= max_pending`, the oldest *unacknowledged* packet is `pop_front`-ed; subsequent NACK can never recover that seq because the entry is gone (`on_nack` at 408-417 finds nothing to match). Fix: backpressure callers; don't drop tracking for in-flight packets.

18. **`adapter/net/router.rs:559-580` (`NetRouter::start`) ‚Äî No re-entry guard.** Calling twice spawns two competing dequeue loops on the same `FairScheduler`, racing `pop()`. Fix: `compare_exchange` on `running`.

19. **`adapter/net/behavior/safety.rs:1378-1407` ‚Äî AuditOnly mode silently drops violation logs.** `check_rate_limits` only logs the global-RPM violation when `mode == Enforce`; AuditOnly's documented "log violations but don't block" doesn't actually log. Fix: log unconditionally; gate only the `return Err` on enforce.

20. **`adapter/net/behavior/proximity.rs:316` (`update_from_pingwave`) ‚Äî Worse path overwrites better.** Updates whenever `pw.seq > self.last_seq` regardless of hop count. A flooded high-seq pingwave through a long route demotes the cached direct route. Fix: separate freshness (always take latest seq) from path quality (only update `hops`/`addr`/`latency_us` when `new_hops <= self.hops`).

21. **`adapter/net/subnet/gateway.rs:144-152` ‚Äî `ParentVisible` over-permissive direction.** Predicate accepts both `dest.is_ancestor_of(source)` and `source.is_ancestor_of(dest)`; the second clause leaks parent-region traffic *down* into descendants, contradicting the documented "visible to ancestors" framing. Tests cover only child‚Üíparent and sibling‚Üísibling.

## Medium

22. **`adapter/net/cortex/adapter.rs:122-141` (`wait_for_seq`)** ‚Äî When `start_seq == 0` and no events fold, the `u64::MAX` sentinel + always-true `running` makes `wait_for_seq(0)` block forever. Direct `CortexAdapter` users on an empty file hang.

23. **`adapter/net/cortex/memories/fold.rs:67-86` ‚Äî Asymmetric upsert semantics under `STORED`.** Existing-id store partial-updates (preserves `pinned`/`created_ns`, overwrites the rest); missing-id inserts with `pinned: false`. The `MemoriesAdapter::store` doc reads "Store a new memory" ‚Äî silent overwrite is surprising. Combined with absence of tombstones (`memories/fold.rs:115`), a STORED‚ÜíDELETED‚ÜíSTORED sequence resurrects deleted entries.

24. **`adapter/net/cortex/memories/query.rs:77-86` ‚Äî Empty-vec filter is a footgun.** `Some(vec![])` for `require_any_tag` excludes everything (`any` over empty = false ‚Üí reject); `Some(vec![])` for `require_all_tags` excludes nothing (`all` over empty = true). UI forms emitting empty multi-selects break silently.

25. **`adapter/net/failure.rs:783-805` (`RecoveryManager::on_failure`)** ‚Äî `failed_nodes.insert(...)` overwrites existing `FailedNodeState`, resetting `failed_at` and `retry_count` to 0. Flapping peers never hit `max_retries`. Fix: `entry().or_insert(...)` and bump `retry_count`.

26. **`adapter/net/session.rs:543-565` (`evict_idle_streams`) ‚Äî LRU vs concurrent open race.** `streams.iter().min_by_key(...)` then `remove` is non-atomic. A freshly-opened stream can be torn down between selection and removal. Fix: `remove_if` with a freshness predicate.

27. **`adapter/net/mod.rs:451` ‚Äî Handshake retry has no upper bound.** `tokio::time::sleep(Duration::from_millis(100 * attempt as u64))` over `MAX_HANDSHAKE_RETRIES = 1024` ‚áí ~14 hours total, last attempt sleeps ~102s. Fix: cap the per-attempt sleep at e.g. 5s.

28. **`adapter/net/redex/file.rs:402-462` (`append`) ‚Äî Observer inconsistency.** `next_seq` is bumped before `segment.append`, so a reader between (`len()`, `next_seq()`) sees `len + 1 > next_seq_seen`. Fix: maintain a separate `committed_seq` for observers.

29. **`bus.rs:1337` ‚Äî `JoinHandle` errors silently dropped.** `let _ = futures::future::join_all(drains).await;` ‚Äî a panicked drain worker isn't logged unless the runtime's panic hook is configured (default Tokio doesn't). Stranded events.

30. **`adapter/net/identity/origin.rs:42-43` ‚Äî `origin_hash: u32` collision floor.** ~65k peer birthday-collision; cross-channel accounting keyed by `origin_hash` aliases distinct entities. Fix: rename to `origin_tag` to dispel identity reading, or widen to u64 in a wire bump.

31. **`shard/mapper.rs:1024` (`finalize_draining`) ‚Äî Wrong atomic ordering.** Reads `pushes_since_drain_start` with `Relaxed` while the field's docstring requires `Acquire` to pair with the writer's `SeqCst` reset.

32. **`adapter/net/identity/token.rs:565-594` (`insert_unchecked`) ‚Äî Slot-cap race unbounded.** `contains_key` then `entry()` overshoot is bounded by concurrent calls, not shards. Fix: `entry().or_insert_with()` then drop on overflow.

33. **`adapter/net/behavior/diff.rs:241-243` ‚Äî `to_bytes` returns `Vec::new()` on cap-violation.** Indistinguishable from a legitimate empty diff. Senders silently transmit zero bytes; receiver drops. Fix: deprecate `to_bytes` in favor of `try_to_bytes`.

34. **`adapter/net/contested/propagation.rs:62-106` ‚Äî Lossy `f64 ‚Üí u64` poisons EWMA.** A pathological RTT clamps `per_hop` to `u64::MAX`, locking `base_hop_latency_nanos` near max permanently. Combined with the `multiplier == 0.0` check (line 96) which doesn't catch NaN.

35. **`adapter/net/contested/correlation.rs:148` ‚Äî `Instant` subtraction can panic.** `now - self.config.correlation_window` panics if `correlation_window > now.elapsed()` (configurable to hours).

36. **`adapter/net/contested/superposition.rs:107-111` ‚Äî `target_replayed` only advances from `Superposed`.** If called while in `Spreading` (target catches up before `advance(Replay)`), phase stalls in `Spreading` forever; `ReadyToCollapse` never fires.

37. **`adapter/net/contested/observation.rs:62-73, 115` ‚Äî Unbounded `HashMap` growth + `seq_diff_sum` overflow.** No eviction in `last_observed`/`estimated_delay`; `u64::abs_diff` accumulator overflows on long chains. Fix: LRU + `saturating_add`.

38. **`adapter/net/compute/orchestrator.rs:1177-1182` (`on_replay_complete`) ‚Äî Forged `target_head` with `parent_hash: 0`.** Synthetic link doesn't match any real hash; downstream verifiers can't reconcile.

39. **`adapter/net/compute/migration_target.rs:244-261` (`buffer_event`) ‚Äî No phase guard.** Inserts/delivers post-cutover; combined with normal-path delivery yields duplicate execution.

40. **`adapter/net/compute/migration_source.rs:107-145` (`start_snapshot`) ‚Äî Snapshot side-effect race.** `contains_key` ‚Üí `entry()` window allows two concurrent snapshots of the same origin via user `MeshDaemon::snapshot()`. Trait API doesn't enforce idempotency.

41. **`adapter/net/compute/standby_group.rs:213-253` (`sync_standbys`)** ‚Äî Partial-failure leaves standbys at mixed states with `synced_through` unchanged and buffer not cleared; subsequent `promote()` may double-deliver.

42. **`adapter/net/compute/registry.rs:91-96` (`unregister`)** ‚Äî In-flight `Arc<Mutex<DaemonHost>>` clones survive past unregister; a fast `register` of the same origin produces two parallel Arcs locking independently, splitting writes.

43. **`adapter/net/state/causal.rs:248` ‚Äî Doc says `CausalLink: 24 bytes`, constant is 28.** Wire-format spec contradicts code; external parsers reading per the doc go off-by-4.

44. **`adapter/net/contested/partition.rs:241-246` ‚Äî `NaN >= threshold` blocks healing.** When `other_side.is_empty()`, ratio is NaN and partition can't heal automatically.

45. **`adapter/net/redex/disk.rs:1196` ‚Äî `saturating_sub(dat_base) as u32` for stale heap entry silently writes offset 0.** Currently safe by `dat_base` derivation, but masks corruption rather than failing.

46. **`adapter/net/traversal/portmap/natpmp.rs:537-606` (`remove`)** ‚Äî Silent early returns on `bind`/`connect`/`send` failures; operator never sees that the prior mapping wasn't revoked.

47. **`consumer/merge.rs:384` ‚Äî Per-shard cap clamps cursor advancement.** When `unclamped_per_shard > PER_SHARD_FETCH_CAP`, cursor rolls back instead of advancing to the last fetched event id; single-shard requests with filters can stall in re-fetch loops.

48. **`bus.rs:1245` ‚Äî `std::time::Instant` mixed with tokio time.** Wall-clock `5s` in shutdown breaks `tokio::time::pause()`-based tests.

49. **`adapter/net/route.rs:461-481` ‚Äî Doc/code mismatch on tie-break.** Doc says "preserved if strictly better"; code uses `<=`, allowing equal-metric overwrite.

50. **`ffi/mod.rs:563` (and similar handle accessors) ‚Äî `&*handle` without alignment check.** Misaligned `*mut NetHandle` from C is immediate UB before the null check; only `is_null()` is validated.

## Audit notes

- Three subsystems are notably well-defended: `crypto.rs` (nonce/replay/HKDF/AEAD all sound), `pool.rs` (TX nonce coordination correct), and most `FailureDetector`/`CircuitBreaker` paths.
- `mesh.rs` (~7953 lines) was not fully read in this pass ‚Äî only spot-checks. Worth a dedicated follow-up.
- The codebase has aggressive regression tests (`test_regression_*`) that pin many historical bugs; the findings above are not covered by those.
- Triage priority: #1, #2, #3 (Critical), then #4 (temp-file leak), #5‚Äì#7 (cortex watermark/checksum durability), #12 (`registry::replace`), #15 (FFI DoS).
